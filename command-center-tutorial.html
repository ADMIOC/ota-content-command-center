Before the first campaign

Creator operating rules

These rules make the platform adoptable by a full internal team. They define how creators behave inside the tool before any single production step begins.

Rule 1

Do not start with files.

Start with brand, campaign, creative direction, and stage state. Files come after the workstream knows what it is producing.

Rule 2

Do not bypass the co-pilot.

Use the OTA Co-Pilot for next steps, draft actions, and on-brand suggestions. The guide is part of the product experience.

Rule 3

Do not treat approval as informal.

Approvals must be visible in the approval gate, stage status, notes, and stored evidence before handoff.

Rule 4

Do not hide improvement requests.

Use the section-specific review lane so feedback becomes a GitHub-tracked issue instead of a forgotten chat message.

Section-by-section tutorial

How creators use every Command Center section

Each section below explains the purpose, strategy, tactical action, and completion standard. Creators should use this as the official training reference during onboarding and daily work.

01

Top bar

Command Center entry point

Purpose: Give creators one launch point for new campaigns, brand management, exports, demo resets, and training pages.

Strategy: Reduce decision fatigue. The top bar keeps the core operating actions visible at all times.

Tactical use: Use New Campaign to start work, Brands to manage brand intelligence, Export to preserve local workspace state, and Tutorial or Creator Journey for training.

Done when: The creator knows where to start, where to learn, and where to export a workspace snapshot.

02

Sidebar

Campaign queue, metrics, search, and stage filter

Purpose: Show active work, approved work, blocked work, and the current campaign queue.

Strategy: Creators should choose work by operational priority, not memory or chat history.

Tactical use: Search by campaign or brand, filter by stage, and select the campaign card that needs action today.

Done when: The correct campaign is selected and the creator understands its readiness, status, owner, and due date.

03

Campaign overview

Header, readiness score, and campaign metadata

Purpose: Make the campaign's identity and completion health visible before any detailed work begins.

Strategy: A creator should never wonder whether a campaign is moving, stuck, or ready.

Tactical use: Check the brand, campaign name, platform, output quantity, owner, due date, and readiness score before editing any section.

Done when: The campaign overview tells the same story as the detailed stages below it.

04

Brand intelligence

Active Brand Profile and Manage Brand Profile

Purpose: Make brand voice, audience, offers, cinematic perspective, compliance posture, and platform preferences reusable.

Strategy: Brand profile is the source of truth. Every script, scene, caption, and package inherits it.

Tactical use: Open Brands or Edit Brand, select the profile, use the OTA Brand Build Co-Pilot, draft missing fields, and save only after the profile is specific enough for production.

Done when: The brand profile can guide a creator who has never worked on that brand before.

05

Launch Workstream

Campaign setup dialog

Purpose: Start a campaign with the correct brand, campaign name, platform, output count, owner, due date, creative direction, and guardrails.

Strategy: Campaign setup is the contract for the entire production chain. Weak setup creates weak scenes, weak scripts, and rework.

Tactical use: Choose an active brand from the dropdown or Other for a new brand, review the agentic creative direction, update it with human judgment, and confirm regulated-brand guardrails when required.

Done when: The workstream has a clear creative direction and every required field is complete before scene planning begins.

06

Agentic creative direction

Creative Direction Versions

Purpose: Store the campaign's storytelling lens and keep track of direction changes.

Strategy: Direction should define point of view, emotional stakes, visual rhythm, audio-led pacing, and repeatable scene language.

Tactical use: Use Regenerate when the direction is too generic, Use Version when a prior direction is stronger, and preserve creator edits in the active campaign direction.

Done when: The active direction is specific enough to shape scripts, visuals, captions, and production handoffs.

07

OTA Co-Pilot

Live Build Assistance

Purpose: Guide creators through the next logical action in each section and dialog.

Strategy: The co-pilot is the hallmark of the OTA Social Engine experience. It should reduce blank-page thinking and keep decisions on-brand.

Tactical use: Read the guidance, apply draft actions when useful, then edit the result. The creator remains accountable for final judgment.

Done when: The creator knows the next move, the suggested draft has been reviewed, and no section is being completed blindly.

08

Review lane

Section Improvement Requests

Purpose: Turn feedback into targeted, trackable improvement work.

Strategy: Feedback should attach to the exact section it improves, with priority and reviewer context.

Tactical use: Pick the section, enter reviewer name, set priority, write the improvement request, then copy it or open the GitHub issue draft.

Done when: The improvement has a section, priority, context, and a path into GitHub tracking.

09

Stage rail

Ten-stage workflow navigation

Purpose: Move every campaign through the same production sequence: Setup, Scenes, ElevenLabs, Remotion, Higgsfield, QA, Assembly, Package, Bunny, and Blotato.

Strategy: Standardized stages create predictable work, easier handoff, and measurable readiness.

Tactical use: Click the current stage, update status, assign owner, set due date, complete checklist items, and add notes.

Done when: Stage status, checklist, owner, due date, and notes reflect real production state.

10

Scene Queue

Scene planning and prompt control

Purpose: Convert a broad campaign into scene-level jobs with titles, scripts, prompts, compliance notes, and statuses.

Strategy: Scene planning makes production modular. Each scene should be specific enough for audio, Remotion, and Higgsfield work.

Tactical use: Click Add Scene, use the co-pilot to draft fields, write the video script first, then shape the Higgsfield prompt around that audio-led spine.

Done when: Each scene has a script, visual prompt, compliance note if needed, and an accurate generation status.

11

ElevenLabs

Video script and audio tracks

Purpose: Approve scripts and audio tracks before the Remotion assembly pass.

Strategy: Audio comes before Higgsfield. The successful Codex + Remotion workflow ingests ElevenLabs output, then Higgsfield ingests Remotion output.

Tactical use: Confirm the ElevenLabs account, API status, voice profile, script URL, audio tracks URL, and notes. Use Copy Brief for handoff context.

Done when: The approved script/audio package is ready for Codex + Remotion and no voice assets are missing.

12

Codex + Remotion

Audio-led assembly pass

Purpose: Turn ElevenLabs script/audio output into the structured production input Higgsfield Studio consumes.

Strategy: Remotion aligns timing, captions, pacing, and composition so visual generation is not guessing from loose notes.

Tactical use: Enter source audio URL, Remotion output URL, and composition notes. Use Copy Brief for the production handoff.

Done when: Higgsfield has an approved Remotion package and the Command Center stores the source and output references.

13

Higgsfield

Visual generation stage

Purpose: Generate clips from the approved Remotion input and separate usable takes from failed generations.

Strategy: Higgsfield is production generation, not the source of strategy. It should follow the approved audio-led package.

Tactical use: Track generation files, raw outputs, failed takes, usable clips, and revision notes in the stage checklist and notes.

Done when: Usable clips are promoted to review and failed generations are visible for learning and revision.

14

Approval Gate

Human QA and final checks

Purpose: Keep publishing approval human-led and explicit.

Strategy: Automation can assist, but public content requires visible quality, compliance, caption, storage, and package approval.

Tactical use: Complete QA checks, add notes, update stage status, and use comments for any unresolved concerns.

Done when: The approval gate shows the campaign is safe to assemble, store, package, and publish.

15

Bunny Storage

Central asset archive

Purpose: Store approved media and production evidence in one clean media-library layer.

Strategy: Storage is the bridge between production and publishing. Every final asset needs a durable URL.

Tactical use: Add the folder path, final video URL, thumbnail URL, script URL, ElevenLabs audio URL, Remotion output URL, caption doc, scene files, and approval doc.

Done when: Copy Manifest produces a complete storage record another teammate can use without asking where files live.

16

Publishing Package

Captions, hashtags, and Blotato handoff

Purpose: Bundle the approved media with the exact copy needed for publishing.

Strategy: Publishing should not require reinterpreting the campaign. The package should be clear enough to schedule directly.

Tactical use: Write the caption, hashtags, and platform notes. Confirm media URLs are approved, then use Copy Package for Blotato or future publisher integration.

Done when: The publisher has approved media, caption, hashtags, platform notes, and storage references in one package.

17

Activity Log and export

Operational memory

Purpose: Preserve what agents and creators changed during campaign production.

Strategy: A scalable content engine needs a visible audit trail and portable workspace snapshot.

Tactical use: Review the Activity Log, copy it when sharing progress, and export the workspace JSON when a snapshot is needed.

Done when: Another teammate can understand campaign history, status, and files without a meeting.

Daily tactical workflow

The creator checklist for every campaign

  1. Open the Command Center and select the campaign from the queue.
  2. Read the overview, readiness score, active stage, owner, and due date.
  3. Confirm the brand profile is specific and current.
  4. Review the OTA Co-Pilot guidance before editing the section.
  5. Complete the current stage checklist and update status honestly.
  6. Add or refine scenes with script-first, audio-led planning.
  7. Move ElevenLabs script and audio into Codex + Remotion before Higgsfield.
  8. Run QA, approval, storage, and publishing package checks before handoff.
  9. Use section comments and GitHub issue drafts for improvements.
  10. Export or copy logs when the campaign needs a portable record.
